# Security Review

Perform security review for: $ARGUMENTS

## Steps

1. **Identify Attack Surface**
   - Entry points
   - Data flows
   - External dependencies

2. **Check Vulnerabilities**
   - Injection attacks
   - Authentication issues
   - Authorization flaws
   - Data exposure

3. **Review Configuration**
   - Security headers
   - HTTPS usage
   - Secret management

4. **Document Findings**
   - Categorize by severity
   - Provide remediation
   - Track follow-ups

## Output

Provide:
- Vulnerability assessment
- Findings by severity
- Remediation steps
- Recommendations
