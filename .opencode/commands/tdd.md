# TDD Workflow

Implement using Test-Driven Development for: $ARGUMENTS

## Steps

1. **Write Failing Test**
   - Define expected behavior
   - Write minimal test
   - Verify it fails

2. **Write Minimal Code**
   - Implement just enough to pass
   - Don't over-engineer
   - Verify test passes

3. **Refactor**
   - Improve code quality
   - Remove duplication
   - Verify tests still pass

4. **Repeat**
   - Add next test case
   - Continue cycle

## Output

Provide:
- Test plan
- Test cases with inputs/outputs
- Implementation code
- Coverage results
